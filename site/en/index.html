What is OpenODC

OpenODC is an open-source platform for defining, comparing, and browsing ODC (Operational Design Condition) declarations for Automated Driving Systems. It is rigorously aligned with the Chinese national standard GB/T 45312—2025 Intelligent and connected vehicles — Operational design condition for automated driving system.

① ODC Schema

The full GB/T 45312—2025 hierarchy as a machine-readable JSON Schema and TypeScript types — about 80 fifth-level atomic elements and all quantitative scales (12 wind levels, 4 rain levels, visibility tiers, etc.).

② ODC Editor

OEMs or community contributors author standardized ODC declarations through a hierarchical web form, with live preview of the generated table and one-click export to JSON / Markdown / Excel / PDF.

Phase 1

③ ODC Gallery

A public sample library — browse, filter, and compare ODCs by vehicle, function, automation level, and vendor. Each record carries transparent provenance: official, community-extracted, or media-tested.

Phase 1–2

④ Multi-View Renderer

One ODC document, four lenses: developer (full hierarchy + JSON), tester (scenario sampling grid), regulator (compliance gap matrix), consumer (plain-language permission cards).

Phase 2

Why this exists

Three industry realities

  • Each OEM declares ODCs in its own format — regulators and third-party labs cannot make like-for-like comparisons
  • Consumers can't see the actual operating boundary — sales pitches stop at "L2" or "L3"; the limits surface only after an incident
  • GB/T 45312—2025 defines the full hierarchy, but the standard itself ships no machine-readable counterpart

OpenODC's position

Not a replacement for an OEM's internal ODC tooling — a public-facing standardized format and a public sample library.

Analogy: caniuse.com for browser features, applied to operational design conditions. Make ODCs queryable, comparable, and reusable.

Open data first; expect OEMs to follow once the format becomes the industry default.

Current status (v0.1.0 · Phase 0)

  • Full transcription of the GB/T 45312—2025 ODC hierarchy (~80 fifth-level elements)
  • JSON Schema (Draft 2020-12) and TypeScript type definitions
  • All quantitative scales captured as reusable enums (wind, rain, snow, visibility for fog/haze/dust/smoke, natural and artificial lighting)
  • Standard Appendix A L3 expressway example fully transcribed to JSON
  • Apache 2.0 (code) + CC BY 4.0 (data) licenses
  • Web editor (Phase 1)
  • Public sample gallery (Phase 1)
  • Four-view renderer (Phase 2)
  • Vendor self-service authoring + Supabase backend (Phase 4)

How to participate

For OEMs and AD suppliers

Submit an officially-endorsed ODC declaration for one of your vehicles or functions. We mark it vendor_confirmed and rank it as the authoritative entry for that record.

For third parties, researchers, journalists

Extract ODCs from public materials (owner manuals, regulatory filings, third-party tests). Mark provenance honestly. Submit via PR to enter the community_reviewed tier.
