这是什么

OpenODC 是一个开源的 ODC（Operational Design Condition，设计运行条件）标准化定义、对比、查阅平台。它基于国家标准 GB/T 45312—2025《智能网联汽车 自动驾驶系统设计运行条件》。

① ODC Schema

把 GB/T 45312—2025 的 5 层级元素结构、约 80 个第 5 层原子元素、所有量化分级表（风力 12 级、雨量 4 级、能见度 4 级等）转录为机器可读的 JSON Schema 与 TypeScript 类型。

② ODC Editor

厂家或社区可以通过 Web 表单层级化勾选、输入参数范围，实时预览生成的标准化 ODC 表格。一键导出 JSON / Markdown / Excel / PDF，对接监管申报或仿真平台。

Phase 1

③ ODC Gallery

公开的 ODC 样例库 —— 按车型、功能、自动化等级、厂家筛选浏览。多车型并排对比（diff 视图）。每条数据透明标注来源（厂家声明 / 社区反推 / 官方公告）。

Phase 1–2

④ Multi-View Renderer

同一份 ODC 数据，四种视图：开发者（完整 5 层级 + JSON）、测试（场景生成清单）、监管（合规对照表）、消费者（一句话能/不能用 + 风险卡片）。

Phase 2

为什么要有这个

三个行业现状

  • OEM 各自定义 ODC，没有统一格式 —— 监管和第三方机构难以横向对比
  • 消费者完全看不到具体边界 —— 销售只说"L2/L3"，事故发生时才发现"原来不能用"
  • 国家标准 GB/T 45312—2025 给了完整层级结构，但缺少机器可读的对应物

OpenODC 的定位

不是要取代厂家自己的 ODC 文档系统，而是建立一个面向对外发布的标准化格式 + 公开样例库。

类比：caniuse.com 之于 web 浏览器特性。让设计运行条件变得可查、可比、可复用。

开放数据先行，等待 OEM 跟进。

当前进展（v0.1.0 · Phase 0）

  • 完整转录 GB/T 45312—2025 的 ODC 元素层级（约 80 个第 5 层级元素）
  • JSON Schema 主文件 + TypeScript 类型定义
  • 量化分级表全部机器可读化（风力、雨量、降雪、积雪、4 类能见度、自然/人工光照）
  • 标准附录 A 的 L3 高速 ODC 示例完整转录为 JSON
  • Apache 2.0 代码许可 + CC BY 4.0 数据许可
  • Web 编辑器（Phase 1）
  • 公开样例库 Gallery（Phase 1）
  • 四视图渲染（Phase 2）
  • 厂家直填能力 + Supabase 后端（Phase 4）

参与方式

对 OEM / AD 供应商

提交贵方某个车型 / 功能的官方 ODC 声明。我们会标记为 vendor_confirmed，作为该条目的权威版本。

对第三方机构 / 学术界 / 媒体

从公开材料（车主手册、媒体测评、申报公告）反推 ODC 数据，标注来源置信度。提交 PR 后进入 community_reviewed 状态。
